## Visão Geral

D-Nutri é um aplicativo de rastreamento fitness que ajuda os usuários a registrar e monitorar sua ingestão de calorias, acompanhar o consumo de macronutrientes e manter hábitos alimentares saudáveis.

## Principais Funcionalidades

- **Registro de Refeições**: Permite aos usuários adicionar refeições com detalhes nutricionais completos, incluindo calorias, macronutrientes e vitaminas.
- **Acompanhamento de Metas**: Usuários podem definir metas personalizadas de ingestão diária e acompanhar seu progresso ao longo do tempo.
- **Escaneamento de Código de Barras**: Entrada rápida de alimentos através da leitura de códigos de barras para obter dados nutricionais de um banco de dados.
- **Visualização de Dados**: Fornece gráficos interativos e insights para ajudar os usuários a entenderem suas tendências alimentares.
- **Recomendações Personalizadas**: Sugestões de refeições baseadas em preferências e necessidades alimentares, utilizando inteligência artificial.

## Tecnologias Utilizadas

- **React Native**: Desenvolvimento multiplataforma para iOS e Android.
- **Node.js**: Backend para desenvolvimento de APIs e processamento de dados em tempo real.
- **MongoDB**: Armazenamento de perfis de usuários, registros alimentares e análises.
- **Express.js**: Framework REST API para gerenciamento da lógica de backend.

## Desafios e Aprendizados

Um dos maiores desafios foi garantir cálculos nutricionais precisos e integrar um vasto banco de dados de alimentos mantendo a performance do aplicativo. A equipe também refinou a experiência do usuário simplificando a entrada de alimentos e aprimorando os algoritmos de recomendação.

## Resultado

D-Nutri tem ajudado os usuários a acompanhar suas metas fitness, resultando em uma maior conscientização alimentar. O aplicativo recebeu feedback positivo pela sua facilidade de uso, análises detalhadas e experiência de rastreamento intuitiva.

## Um Novo Capítulo na Minha Jornada de Aprendizado

O desenvolvimento de software é um processo de aprendizado contínuo, repleto de novos conceitos, desafios e descobertas. Recentemente, tenho me aprofundado em **padrões de projeto**, um tema que aprimorou significativamente minha compreensão sobre arquitetura de software e boas práticas.

## Compreendendo o Poder dos Padrões de Projeto

Os padrões de projeto fornecem **soluções comprovadas** para problemas recorrentes no design de software, permitindo que os desenvolvedores escrevam códigos **mais escaláveis, fáceis de manter e eficientes**. Antes de estudá-los, frequentemente me via **repetindo código, criando componentes fortemente acoplados e enfrentando dificuldades na manutenção a longo prazo**. Aprender esses padrões mudou a forma como **estruturo e organizo minhas aplicações**.

## Aplicando Padrões no Meu Trabalho

Como desenvolvedor backend que trabalha com **Node.js e TypeScript**, venho incorporando **padrões de criação**, como:

- **Singleton**: Gerencia uma única instância de um recurso em toda a aplicação.
- **Factory**: Cria objetos dinamicamente enquanto reduz dependências.
- **Builder**: Constrói objetos complexos passo a passo com maior flexibilidade.

Esses padrões **melhoraram a estrutura do meu código**, tornando-o **mais fácil de escalar e manter**. Além disso, ajudaram-me a **escrever código mais limpo e reutilizável**, reduzindo a complexidade em aplicações reais.

## Superando Desafios

Entender **quando e onde aplicar padrões de projeto** foi um desafio. No início, me peguei **usando padrões em excesso**, tentando encaixá-los em lugares onde não eram necessários. Com o tempo, aprendi que **o segredo é saber quando um padrão agrega valor real**, em vez de usá-lo apenas por conveniência.

## Principais Lições do Meu Aprendizado

- **Padrões de projeto não são regras, são diretrizes** – Eles ajudam a estruturar o código, mas devem ser usados **apenas quando resolvem um problema real**.
- **Legibilidade e manutenção são essenciais** – Um projeto bem estruturado facilita a entrada de novos desenvolvedores e a depuração do código.
- **A prática é essencial** – Aprender sobre padrões de projeto na teoria é útil, mas **implementá-los em cenários reais é o que realmente faz a diferença**.

## O Que Vem a Seguir?

Com uma base sólida em **padrões de criação**, meu próximo passo é **explorar padrões estruturais e comportamentais**, aplicando-os à minha **arquitetura backend** para melhorar **reutilização e flexibilidade do código**.

## Visão Geral

D-TRACKER é uma ferramenta exclusiva para gerenciamento de projetos de mapeamento, projetada para ajudar gestores a supervisionar dados fornecidos por editores e revisores. O sistema permite que gestores definam áreas do projeto, carreguem dados de suporte, cadastrem usuários, atribuam áreas de trabalho e extraíam estatísticas, melhorando a eficiência e a qualidade das tarefas colaborativas de mapeamento.

## Principais Funcionalidades

- **Gerenciamento de Áreas do Projeto**: Permite que gestores delimitem áreas de mapeamento e definam zonas de trabalho.
- **Gestão de Usuários**: Cadastro e gerenciamento de usuários como editores e revisores.
- **Atribuição de Áreas de Trabalho**: Designação de áreas de mapeamento para os editores processarem dados.
- **Mapeamento de Polígonos**: Edição, criação e validação de polígonos mapeados.
- **Correção de Erros**: Revisores analisam dados, reportam erros e garantem a precisão do mapeamento.
- **Análise Estatística**: Geração de relatórios detalhados filtrados por usuário e período de tempo.
- **Exportação de Relatórios**: Possibilidade de exportar relatórios estatísticos em formato PDF.
- **Autenticação de Usuários**: Controle de acesso seguro ao sistema.

## Tecnologias Utilizadas

- **React TypeScript**: Desenvolvimento frontend.
- **Node.js com TypeScript**: Backend do sistema.
- **MongoDB**: Banco de dados NoSQL para armazenamento de dados de mapeamento.
- **Express.js**: Desenvolvimento de APIs REST.
- **SCRUM**: Metodologia ágil para gerenciamento do projeto.
- **Ferramentas de Gestão**: Uso de Trello ou Jira para rastreamento de tarefas.

## Desafios e Aprendizados

O principal desafio foi estruturar o gerenciamento de dados de mapeamento e garantir um fluxo de trabalho eficiente entre editores, revisores e gestores. A equipe aprendeu a aplicar conceitos avançados de POO, padrões de projeto e testes de software para desenvolver um sistema robusto e escalável.

## Resultado

D-TRACKER conquistou o **1º lugar** na competição entre todas as turmas do curso DSM da FATEC Jacareí. O sistema se mostrou essencial para a gestão colaborativa de mapeamento, otimizando processos e centralizando dados de forma eficiente.

## Visão Geral

Bikes4you é uma aplicação web peer-to-peer projetada para conectar proprietários de bicicletas com pessoas interessadas em alugá-las. A plataforma atua exclusivamente como intermediária, facilitando a conexão entre as partes sem gerenciar pagamentos.

## Principais Funcionalidades

- **Listagem de Bicicletas**: Proprietários podem listar suas bicicletas, definir disponibilidade e preços de aluguel.
- **Solicitação de Aluguel**: Locatários podem navegar pelas bicicletas disponíveis e solicitar aluguéis.
- **Autenticação de Usuário**: Segurança reforçada com autenticação JWT.
- **Comunicação Direta**: Mensagens entre proprietários e locatários.
- **Gerenciamento de Disponibilidade**: Atualização em tempo real da disponibilidade das bicicletas.
- **Sem Gestão de Pagamentos**: A plataforma apenas facilita a conexão, sem responsabilidade sobre os pagamentos.

## Tecnologias Utilizadas

- **TypeScript**: JavaScript fortemente tipado para maior confiabilidade.
- **Node.js**: Framework backend para desenvolvimento de APIs.
- **PostgreSQL**: Banco de dados relacional para gerenciamento de dados.
- **TypeORM**: ORM para operações no banco de dados.
- **React**: Framework frontend para UI interativa.
- **Styled Components**: CSS-in-JS para estilização dinâmica.
- **Material UI**: Componentes pré-construídos para design consistente.
- **JWT**: Mecanismo seguro de autenticação.

## Desafios e Aprendizados

Um dos maiores desafios foi garantir uma experiência de usuário fluida, mantendo segurança e integridade dos dados. A equipe focou em otimizar consultas ao banco de dados com TypeORM, garantir autenticação segura com JWT e aprimorar a interface com Material UI e Styled Components.

## Resultado

Bikes4you conectou com sucesso proprietários de bicicletas e locatários, criando um processo de aluguel eficiente e seguro. O projeto demonstrou o potencial do compartilhamento peer-to-peer na mobilidade urbana e foi bem recebido pelos usuários pela facilidade de uso e confiabilidade.
